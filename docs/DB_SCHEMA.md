# Схема базы данных «Битва Дашбордов»

Документ описывает таблицы и связи развёртываемой БД (Liquibase). Связь сущностей БД с бизнес-сущностями — в конце.

---

## Таблицы

### users

Пользователь веб-приложения (регистрация, вход).

| Колонка        | Тип         | Описание                    |
|----------------|-------------|-----------------------------|
| id             | BIGSERIAL   | Первичный ключ              |
| email          | VARCHAR(255)| Уникальный email            |
| password_hash  | VARCHAR(255)| Хэш пароля                  |
| role           | VARCHAR(50) | Роль (по умолчанию USER)    |
| created_at     | TIMESTAMP   | Дата создания               |

---

### companies

Компания, привязанная к пользователю. К компании привязываются интеграции МойСклад и Telegram.

| Колонка    | Тип         | Описание           |
|------------|-------------|--------------------|
| id         | BIGSERIAL   | Первичный ключ     |
| name       | VARCHAR(255)| Название компании  |
| user_id    | BIGINT      | Внешний ключ на users(id), ON DELETE CASCADE |
| created_at | TIMESTAMP   | Дата создания       |

---

### moysklad_integrations

Подключение аккаунта МойСклад к компании (токен хранится в зашифрованном виде).

| Колонка         | Тип         | Описание                          |
|------------------|-------------|-----------------------------------|
| id               | BIGSERIAL   | Первичный ключ                    |
| company_id       | BIGINT      | Внешний ключ на companies(id), CASCADE |
| token_encrypted  | TEXT        | Зашифрованный токен API           |
| status           | VARCHAR(50) | Статус (по умолчанию ACTIVE)       |
| created_at       | TIMESTAMP   | Дата создания                     |

---

### telegram_integrations

Подключение Telegram-канала к компании (токен бота в зашифрованном виде).

| Колонка             | Тип         | Описание                          |
|---------------------|-------------|-----------------------------------|
| id                  | BIGSERIAL   | Первичный ключ                    |
| company_id          | BIGINT      | Внешний ключ на companies(id), CASCADE |
| bot_token_encrypted | TEXT        | Зашифрованный токен бота          |
| channel_chat_id     | VARCHAR(100)| Идентификатор канала              |
| status              | VARCHAR(50) | Статус (по умолчанию ACTIVE)      |
| created_at         | TIMESTAMP   | Дата создания                     |

---

### top_n_reports

Сформированный отчёт ТОП-N по компании за период. Статус: DRAFT → APPROVED → PUBLISHED.

| Колонка      | Тип       | Описание                              |
|--------------|-----------|---------------------------------------|
| id           | BIGSERIAL | Первичный ключ                        |
| company_id   | BIGINT    | Внешний ключ на companies(id), CASCADE |
| period_start | DATE      | Начало периода                        |
| period_end   | DATE      | Конец периода                        |
| status       | VARCHAR(50)| DRAFT / APPROVED / PUBLISHED          |
| created_at   | TIMESTAMP | Дата создания                         |
| published_at | TIMESTAMP | Дата публикации (например, в Telegram) |

---

### top_n_entries

Строка рейтинга в отчёте ТОП-N: сотрудник, выручка, маржа, «любимый» товар, место.

| Колонка          | Тип          | Описание                            |
|-------------------|--------------|-------------------------------------|
| id                | BIGSERIAL    | Первичный ключ                      |
| report_id         | BIGINT       | Внешний ключ на top_n_reports(id), CASCADE |
| employee_name     | VARCHAR(255) | Имя сотрудника                      |
| employee_id_ms   | VARCHAR(100) | ID сотрудника в МойСклад           |
| revenue           | NUMERIC(15,2)| Выручка                             |
| margin            | NUMERIC(15,2)| Маржа                               |
| favorite_product  | VARCHAR(255) | «Любимый» товар лидера              |
| rank              | INT          | Место в рейтинге                    |

---

## Связь сущностей БД с бизнес-сущностями

| Таблица БД               | Бизнес-сущность      | Описание |
|--------------------------|----------------------|----------|
| users                    | User                 | Пользователь приложения: регистрация, авторизация. |
| companies                | Company              | Компания пользователя; к ней привязываются интеграции и отчёты. |
| moysklad_integrations    | MoySkladIntegration  | Привязка компании к аккаунту МойСклад (токен, статус). |
| telegram_integrations    | TelegramIntegration  | Привязка компании к Telegram-каналу (бот, канал). |
| top_n_reports            | TopNReport           | Сформированный отчёт ТОП-N за период (черновик → одобрен → опубликован). |
| top_n_entries            | TopNEntry            | Строка рейтинга: сотрудник, метрики, место. |

Служебная таблица Liquibase: `databasechangelog` — учёт выполненных миграций (не трогать вручную).
