# Битва Дашбордов

**Программный комплекс для анализа производительности персонала, занятого в продажах, на предприятиях под управлением CRM «МойСклад» и репрезентации статистики во внешних информационных системах.**

---

## О проекте

Проект разрабатывается в рамках **производственной практики** (12 недель, начиная с 11.02.2026). Цель — реализовать работающее приложение, которое:

- подключается к аккаунтам пользователей в облачной ERP **МойСклад**;
- собирает метрики продаж по сотрудникам (выручка, прирост, маржа и т.п.);
- формирует **ТОП-N** самых успешных продавцов за выбранный период;
- публикует одобренную сводку в **Telegram-канал** для мотивации команды и бенчмаркинга.

Идея близка к формату «Exit Porn» для стартапов: наглядное сравнение результатов и возможность учиться у лидеров.

---

## Целевая аудитория

- Владельцы магазинов и интернет-магазинов
- Логисты и менеджеры по продажам на маркетплейсах (Ozon, Wildberries, Яндекс Маркет и др.)
- Компании, ведущие учёт в **МойСклад** и заинтересованные в мотивации сотрудников через открытую статистику

---

## Ценность продукта

- **Мотивация** — публичное сравнение результатов по объективным метрикам
- **Бенчмаркинг** — понимание, кто лидер и по каким показателям
- **Обратная связь** — возможность указывать «любимый» товар лидера по прибыли и давать советы
- **Прозрачность** — единое место (Telegram-канал) для отображения успехов команды

## Архитектура (целевой стек)

| Слой | Технологии |
|------|------------|
| **Фронтенд** | Node.js (Express) + React (Redux, Redux Saga), REST для UI |
| **Бэкенд** | Java, Spring Boot (Spring MVC), REST API, JSON |
| **БД** | PostgreSQL (для MVP допустим SQLite), Liquibase или SQL-скрипты для миграций |
| **Интеграции** | МойСклад JSON API, Telegram Bot API (канал для публикаций) |
| **Инфраструктура** | Docker, Docker Compose, Nginx, Coolify (развёртывание) |
| **Тесты** | Unit-тесты (логика, сервисы, репозитории, конвертеры), UI-тесты (Selenium), функциональное и нагрузочное тестирование (с учётом лимитов сторонних API) |

Пользователь взаимодействует с системой через **веб-интерфейс** (регистрация, настройка интеграций, создание и одобрение ТОП-N). Публикация результатов — в **Telegram-канал** (управленческий Telegram-бот в текущей реализации может не входить в MVP).

## Внешние системы

- **МойСклад** — источник данных о продажах и сотрудниках (JSON API, токен пользователя).
- **Telegram** — канал для публикации одобренных ТОП-N; при необходимости — бот для управления.

Документация API МойСклад: https://dev.moysklad.ru/

---

## Ключевые сущности и сценарии

- **Пользователь** — регистрация, авторизация, привязка интеграций.
- **Интеграция с МойСклад** — CRUD: создание/обновление/удаление/чтение (токен, параметры доступа).
- **Интеграция с Telegram** — CRUD для привязки каналов/бота.
- **Компания** — ассоциация с аккаунтами МойСклад и Telegram-каналами.
- **ТОП-N (TopN DTO)** — создание на основе данных МойСклад → одобрение/отклонение → публикация в канал или отмена.

Подробное описание бизнес-сущностей, запросов/ответов и структуры БД — в документе [ОПИСАНИЕ ПРОЕКТА И ПРАКТИКИ](./docs/ОПИСАНИЕ_ПРОЕКТА_И_ПРАКТИКИ.md).

## Запуск проекта (разработка и проверка)

### Требования

- **Docker** и **Docker Compose** — для запуска в контейнерах (PostgreSQL + бэкенд).
- **JDK 17+** и **Maven** — для локальной сборки и запуска бэкенда без Docker.

### Клонирование

```bash
git clone https://github.com/MolotilkaHolotilka/dashboard_battle.git
cd dashboard_battle
```

### Запуск через Docker (рекомендуется)

Из корня проекта:

```bash
docker compose up -d
```

Или скрипты из папки `scripts/`:

- **Windows:** `scripts\start.bat`
- **Linux / macOS:** `./scripts/start.sh` (при необходимости: `chmod +x scripts/start.sh`)

Поднимаются сервисы:

- **PostgreSQL** — порт `5432`, БД `dashboard_battle`, пользователь/пароль `postgres`.
- **Бэкенд (Spring Boot)** — порт `8080`.

Проверка работы бэкенда: открыть в браузере или выполнить запрос:

- [http://localhost:8080/actuator/health](http://localhost:8080/actuator/health)

Подробнее: [docs/LOCAL_RUN.md](docs/LOCAL_RUN.md) (подключение к БД, просмотр структуры). Листинг скриптов запуска: [docs/SCRIPTS.md](docs/SCRIPTS.md).

### Запуск без Docker (локально)

1. Установить и запустить PostgreSQL. Создать базу `dashboard_battle` и пользователя (например, `postgres` / `postgres`).
2. В корне проекта перейти в папку бэкенда и запустить:

```bash
cd backend
mvn spring-boot:run
```

По умолчанию приложение подключается к `jdbc:postgresql://localhost:5432/dashboard_battle` с логином и паролем `postgres`. Переменные окружения `SPRING_DATASOURCE_URL`, `SPRING_DATASOURCE_USERNAME`, `SPRING_DATASOURCE_PASSWORD` переопределяют настройки.

При первом запуске Liquibase создаёт таблицы в БД.

### Запуск на другой системе

Требования те же: Docker и Docker Compose (или JDK 17+, Maven и локальный PostgreSQL). Скрипты `start.bat` и `start.sh` рассчитаны на Windows и Linux/macOS.
